# Windows Workload Baseline Toolkit

PowerShell + Python port of the [macOS toolkit](../README.md). Captures CPU/Memory/GPU snapshots every 30 minutes for 1-2 weeks, then assembles a single PDF report.

> **Heads-up:** this Windows version was written end-to-end with code review but has **not** been validated on a real Windows machine yet. If you hit a bug, ping Rick -- fix is usually one PR away.

## Install

Pick **one** method.

### Method 1 -- One-line install (recommended)

Open PowerShell (no admin required) on the Windows machine you want to measure and paste:

```powershell
iwr -useb https://raw.githubusercontent.com/Quattio/workload-baseline/main/windows/install.ps1 | iex
```

This downloads the toolkit to `%LOCALAPPDATA%\Quattio\workload-baseline`, drops a `baseline.cmd` shim into `%USERPROFILE%\.local\bin`, and adds that folder to your user PATH.

**Then open a new terminal window** for the PATH change to take effect.

### Method 2 -- Manual (after cloning or downloading the repo)

```powershell
git clone https://github.com/Quattio/workload-baseline.git
cd workload-baseline\windows
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

Or grab the ZIP from [the GitHub page](https://github.com/Quattio/workload-baseline) (green **Code** button -> **Download ZIP**), extract, then `cd` into `windows\` and run the install command above.

### Verify the install worked

In a **new** terminal:

```powershell
baseline help
```

If you see the help screen, you're set.

## Use

Three commands cover the lifecycle:

```powershell
baseline start     # day 0  -- begin scheduled captures (every 30 min while logged in)
baseline status    # any day -- check it's running, see capture count
baseline build     # day 14 -- assemble the PDF (lands on Desktop, auto-opens)
```

Then to clean up:

```powershell
baseline stop                            # stop further captures (data preserved)
baseline uninstall                       # remove the scheduled task (data preserved)
Remove-Item -Recurse -Force $env:USERPROFILE\Workload-Baseline   # delete captured data
```

## What gets captured

Every 30 minutes while the machine is awake and the user is logged on:

- **CPU**: `\Processor(_Total)\% Processor Time` -- overall busy %
- **Memory**: `\Memory\% Committed Bytes In Use`, `\Memory\Available MBytes`, `\Memory\Pages/sec`
- **GPU**: `\GPU Engine(*engtype_3D)\Utilization Percentage` (Windows 10+, WDDM 2.0). Aggregated across engines. Best-effort -- machines without the counter record `0%`.
- **Top processes**: top 20 by RAM (WorkingSet64) and top 20 by CPU via `Get-Process`
- **Process counts**: Chrome, Edge, Node, Teams, Slack (visibility into "what's running")
- **Per-capture chart**: a PNG showing CPU/GPU/Memory gauges + memory split + process counts, generated by `matplotlib`

Output layout:

```
%USERPROFILE%\Workload-Baseline\
  2026-05-10\
    14-30-report.txt      # text summary
    14-30-data.json       # parsed values
    14-30-chart.png       # headline chart for this capture
```

A typical 14-day campaign produces ~250-350 captures, ~5-20 MB total on disk (Windows version is much smaller than macOS because there are no Activity Monitor screenshots).

### Why no Task Manager screenshots?

The macOS version captures Activity Monitor screenshots via the Quartz window API. Windows Task Manager has no equivalent stable automation path -- driving it via PowerShell SendKeys or UI Automation is flaky across versions and locale settings. Instead, the Windows version generates a per-capture chart PNG (showing the same headline numbers Task Manager would show) and uses those as the visual evidence in the appendix of the final PDF.

If you need raw Task Manager screenshots, the user can take them manually with `Win + Print Screen` and drop them in `%USERPROFILE%\Workload-Baseline\<date>\` -- they will be ignored by the PDF builder but kept as evidence.

## What the PDF contains

Same shape as the macOS version:

1. **Time-series chart** -- memory in use over time (top), GPU% + CPU-busy (bottom). Threshold lines mark `80%` sustained pressure.
2. **Headline stat tiles** -- median memory-committed, % of captures > 80% committed, % > 90% committed (near limit), GPU median/peak, CPU stats.
3. **Appendix** -- every per-capture chart PNG in chronological order.

A typical 14-day campaign produces a 2-8 MB PDF.

## First-run permission prompt

Unlike macOS, no Screen Recording permission dialog appears (no Task Manager screenshots are taken). The first time `baseline start` registers the scheduled task, Windows may ask for elevation if the task runs at a higher integrity level -- our task runs as `Limited` so it should not prompt.

If anti-virus / endpoint protection flags `powershell.exe` executing on a schedule, you may need an allowlist exception. The script is open-source -- send the repo URL to your security team.

## Commands reference

```
baseline start                Install deps + start scheduled captures (every 30 min while logged in)
baseline status               Show task state, capture count, last log line
baseline build                Build PDF from captures, save to %USERPROFILE%\Desktop, open it
baseline build -MinTs T       Skip captures earlier than T (format YYYY-MM-DD_HH-MM)
baseline stop                 Stop further captures (data preserved)
baseline uninstall            Stop + remove scheduled task (data preserved)
baseline help
```

## Dependencies

`baseline start` installs `matplotlib` automatically (via `pip install --user`). You need:

- **Windows 10+** (Windows 11 fully supported)
- **PowerShell 5.1+** (preinstalled). PowerShell 7 also works.
- **Python 3** with `pip` -- install from <https://python.org/downloads/windows/> if missing. Tick "Add Python to PATH" during install.
- **Google Chrome** (or Microsoft Edge -- the build script auto-detects either). Default install paths checked automatically.

## Troubleshooting

**`baseline : The term 'baseline' is not recognized`**
Open a new terminal -- the install adds `%USERPROFILE%\.local\bin` to PATH, but existing windows don't see the change. If still broken: `& "$env:USERPROFILE\.local\bin\baseline.cmd" help`.

**`baseline status` shows the task not running**
Run `baseline start` to register it. If you get a permission error, run from an elevated PowerShell (right-click PowerShell -> Run as administrator).

**Captures land but no `chart.png` appears**
matplotlib isn't installed for the Python interpreter the script is calling. Run `py -m pip install --user matplotlib` and re-run.

**`baseline build` says "Chrome not found"**
The script checks default install paths. If Chrome lives elsewhere, edit `windows/scripts/build_pdf.py` and add the path to `CHROME_CANDIDATES`. Or install Chrome/Edge to a default location.

**Scheduled task is set up but not firing**
- Open Task Scheduler, find `WorkloadBaselineSnapshot`, check its **History** tab.
- Common causes: laptop on battery (we explicitly disable battery-mode runs to mirror macOS launchd-while-awake), user not logged on, or anti-virus blocking PowerShell.

**GPU shows `0%` even under heavy load**
Some integrated GPUs lack the `\GPU Engine\Utilization Percentage` counter. Verify with: `Get-Counter '\GPU Engine(*)\Utilization Percentage'`. If the counter is missing, the rest of the data is still captured.
